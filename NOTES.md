- пользователь просил не терять суть между сессиями;
- важнее доводить проект до рабочего состояния, чем пытаться внедрить все улучшения за один огромный заход.

- серверный слой рабочий по синтаксису;
- БД уже расширена:
  - `users.is_super_admin`
  - `complaints.assigned_admin_id`
  - `complaint_history`
  - `complaint_comments`
- демо-данные уже подготовлены и прогнаны;
- сервер на `3000` в какой-то момент уже был запущен, поэтому повторный старт дал `EADDRINUSE`.

## Что не забыть

### Обычный администратор

- не должен видеть раздел администраторов;
- не должен удалять архив;
- должен работать только с преподавателями, аудиториями, заявками и аналитикой.

### Главный администратор

- видит раздел администраторов;
- может создавать/редактировать/удалять других администраторов;
- может удалять записи из архива.

### Преподаватель

- уведомления только информационные;
- по уведомлению не проваливается в карточку заявки;
- видит только свои аудитории и отправку жалоб.

## Что осталось недожатым

- `admin-room.html` все еще использует старые `prompt()` для части действий;
- `public/js/main.js` перегружен и в будущем его стоит разбить;
- нужно продолжать итеративно, не пытаться снова втащить слишком большой кусок за один раз.

## Быстрый чек перед следующими правками

```powershell
node --check server.js
node --check src\controllers\adminController.js
node --check src\controllers\complaintController.js
node --check public\js\main.js
node scripts\seed-demo-data.js
```

## Если продолжать с того же места

Лучший следующий шаг:

1. привести `admin-room.html` к тому же UX-уровню, что и главную админку;
2. затем полировать карточку заявки;
3. затем уже улучшать графики и дробить `main.js`.
